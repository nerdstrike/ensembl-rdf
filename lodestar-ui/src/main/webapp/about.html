Atlas RDF > Getting started

Overview

To demonstrate the value of the added value of the atlas RDF, we will use experiment E-AFMX-4 as an example. The default Gene Expression Atlas page for E-AFMX-4 is http://www.ebi.ac.uk/gxa/experiment/E-AFMX-4. This provides a nice, human friendly, presentation of information about this experiment. A lot of information is available, like the experiment description, assay and sample information and computed gene expression values. Expression values are further linked to pages about those genes, where you can find more information and further links to external databases such as uniprot and ensembl. This is all good stuff, but suppose you want to do more with this data, such as pulling out specific bits of information or integrating this data with your own application? Well, once again, the Atlas provides access to this data using its REST API, so you can parse XML or JSON, or if you really want the raw data there are TAB delimited files available or even complete relational database dumps that you can use. Every data point in the expression atlas has been assigned a unique URI, for example experiment E-AFMX-4 http://rdf.ebi.ac.uk/resource/atlas/E-AFMX-4

The primary interface to the Expression Atlas RDF data is via the SPARQL endpoint. SPARQL is the standard query language for RDF linked data. You can use our SPARQL endpoint to explore the expression atlas linked data. Some example queries are provides, and a more detailed description of the underlying schema is available here.
